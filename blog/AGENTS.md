# KQode blog agent instructions

## Images

Keep blog article images under `docs/images/`.

- Store each article's images in its own stable English kebab-case topic folder, for example `docs/images/create-rust-project/`.
- Do not include ordering prefixes such as `020-` or translated doc titles in image folder names, because doc order and page titles may change.
- Do not leave IDE-generated screenshots such as `img.png`, `img_1.png`, or `img1.png` in the `docs/` root.
- Rename images to meaningful, English, kebab-case filenames such as `rustrover-welcome-new-project.png`.
- Update Markdown image links after moving images, and keep alt text descriptive.

## Docs ordering

Use numeric filename prefixes for autogenerated sidebar ordering.

- Prefer spaced integer prefixes such as `010-`, `020-`, and `030-` instead of dense `01-`, `02-`, and `03-`.
- Insert new docs between existing docs with the unused gap, for example `015-new-topic.md` between `010-introduction.md` and `020-create-rust-project.md`.
- Use decimal prefixes such as `01.5-...` only when the user explicitly requests an insertion position like `1.5` in an existing dense sequence.
- Keep the human-facing title in frontmatter, so filenames can optimize ordering while page titles stay clean.

## Internationalization

The default blog language is Chinese under `docs/`.

- Translate English content into `i18n/en/docusaurus-plugin-content-docs/current/`.
- Preserve the `docs/` structure and filenames in the English locale tree.
- Preserve routing frontmatter such as `id`, `slug`, and `sidebar_position`; translate only human-facing values such as `title`.
- Preserve code blocks, commands, file paths, URLs, Markdown links, and image paths unless a relative path would break in the English locale.
- Run `cargo xtask blog-build` after changing docs or translations.
