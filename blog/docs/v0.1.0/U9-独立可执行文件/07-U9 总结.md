U9 把 U8 的 ACK demo 推进到本地 standalone artifact。实现 commit 是 [`f642fb3b65bad29da3497d77967af741424205ca`](https://github.com/kefeiqian/KQode/commit/f642fb3b65bad29da3497d77967af741424205ca)：`cargo xtask package` 构建 Rust release backend，再用 Bun compile 生成 `tui/dist/kqode[.exe]`；运行时 materialize 内嵌 backend 到 `~/.kqcode/backends/<version>/<sha256>/`，验证后再启动。

- [`tui/packaged/entry.packaged.tsx`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/tui/packaged/entry.packaged.tsx)：packaged-only composition root。
- [`tui/packaged/embeddedBackendAsset.ts`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/tui/packaged/embeddedBackendAsset.ts)：读取 Bun 内嵌 backend asset。
- [`tui/scripts/buildPackaged.ts`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/tui/scripts/buildPackaged.ts)：staging backend、计算 SHA-256、调用 `Bun.build({ compile })`。
- [`tui/src/bootstrap.ts`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/tui/src/bootstrap.ts)：根据 `__PROD__` 选择 source client 或 packaged client。
- [`tui/src/backend/packaged/`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/tui/src/backend/packaged)：内容寻址 cache、materialize、integrity check 和 packaged launch。

## 关键决策与为什么

1. 单独 packaged entry：因为 Bun-only asset import 不应进入 source / test graph。
2. 使用 Bun `Bun.build({ compile })`：因为它减少 Rollup、Node SEA、`postject` 这类额外移动部件。
3. backend cache 用 `<version>/<sha256>` 内容寻址：因为不同版本和不同内容可以并存，避免覆盖正在运行的 backend。
4. materialization fail closed：因为 packaged 用户不应依赖 Cargo fallback。
5. 保持 `xtask` 入口：因为 KQode 是 Rust-first 项目，Cargo-facing 自动化让 IDE、CI 和命令行一致。

## 有意延后

U9 没有做 GitHub Release、npm、Homebrew、winget、签名、notarization、auto-update 或 daemon 安装；也没有把 ACK demo 升级成真实 agent。它只完成本地 standalone executable 的 runtime shape，为后续发布和真实会话能力打基础。
