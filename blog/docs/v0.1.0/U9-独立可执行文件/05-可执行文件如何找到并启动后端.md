U9 新增 [`createPackagedBackendClient`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/tui/src/backend/client/packagedBackendClient.ts)，但没有重写 JSON-RPC client。它只是把 launch 函数换成 packaged launch，再复用已有 `createBackendClient`。

```ts
export function createPackagedBackendClient(
  options: PackagedBackendClientOptions
): BackendClientHandle {
  return createBackendClient({
    launch: () =>
      launchPackagedBackend({
        asset: options.asset,
        version: options.version,
        workspaceCwd: options.workspaceCwd,
        cacheBaseDir: options.cacheBaseDir
      }),
    requestTimeoutMs: options.requestTimeoutMs
  });
}
```

![packaged client materialize backend 后通过共享 launcher 启动](../../images/u9-standalone-executable/packaged-backend-launch.png)

为什么复用 `createBackendClient`？source mode 和 packaged mode 的协议相同，都是启动一个 Rust stdio backend，然后发送同样的 submit request。差异只在 binary 从哪里来：source mode 通过 Cargo 从源码启动；packaged mode materialize embedded asset 后启动验证过的 binary。

[`launchPackagedBackend.ts`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/tui/src/backend/packaged/launchPackagedBackend.ts) 保持很薄：

```ts
const binaryPath = await materialize({
  asset: options.asset,
  version: options.version,
  cacheBaseDir: options.cacheBaseDir
});

return spawn({
  binaryPath,
  workspaceCwd: options.workspaceCwd
});
```

薄 wrapper 的好处是安全边界集中：materialize 负责 digest 和 cache，已有 `spawnBackend` 负责 env、spawn failure 和 cleanup。packaged path 不重新发明 process 管理。

测试也明确 materialization 失败时不能 spawn：

```ts
await expect(
  launchPackagedBackend(
    { asset, version: '0.1.0', workspaceCwd: '/work/dir' },
    { materialize, spawn }
  )
).rejects.toMatchObject({ kind: BackendErrorKind.Launch });

expect(spawn).not.toHaveBeenCalled();
```

这保证 standalone executable 不会在失败时悄悄改走 source path。
