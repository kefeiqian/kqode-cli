U9 对应 tag [`U9`](https://github.com/kefeiqian/KQode/tree/f642fb3b65bad29da3497d77967af741424205ca)，实现 commit 是 [`f642fb3b65bad29da3497d77967af741424205ca`](https://github.com/kefeiqian/KQode/commit/f642fb3b65bad29da3497d77967af741424205ca)，主题是 `feat(tui): create the standalone kqode executable`。它把 U8 的 source-mode ACK demo 打包成本地 standalone `kqode` 可执行文件。

![U9 standalone kqode 可执行文件结构](../../images/u9-standalone-executable/standalone-executable-overview.png)

计划文档对应 [`U9. Create the standalone kqode executable`](https://github.com/kefeiqian/KQode/blob/17b51456a479ab5de20af403bf1668788099d076/docs/plans/2026-06-25-003-feat-first-ink-tui-homepage-plan.md#u9-create-the-standalone-kqode-executable)。U9 的目标不是 release channel，而是先证明 `tui/dist/kqode[.exe]` 可以直接运行：前端由 Bun 编译成 executable，Rust backend 作为 asset 内嵌，运行时 materialize 到 `~/.kqcode/backends/<version>/<sha256>/` 后再启动。

核心分岔在 [`tui/src/bootstrap.ts`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/tui/src/bootstrap.ts)：

```ts
if (__PROD__) {
  if (loadPackagedAsset === undefined) {
    throw new Error('packaged runtime requires an embedded backend asset loader');
  }
  productVersion = resolveProductVersion({});
  const { createPackagedBackendClient } = await import('@backend/client/packagedBackendClient.ts');
  client = createPackagedBackendClient({
    asset: loadPackagedAsset(),
    version: productVersion,
    workspaceCwd
  });
} else {
  const repoRoot = resolveRepoRoot(path.dirname(fileURLToPath(entryUrl)));
  store.set(repoRootAtom, repoRoot);
  productVersion = resolveProductVersion({ repoRoot });
  const { createSourceBackendClient } = await import('@backend/client/sourceBackendClient.ts');
  client = createSourceBackendClient({ repoRoot, workspaceCwd });
}
```

为什么必须分 source mode 和 packaged mode？开发者路径可以依赖 Cargo 和仓库源码，用户路径不应该。普通用户运行 `kqode` 时，不应被要求安装 Rustup、保留 `node_modules`、进入仓库根目录，或手动启动后端。U9 先把 artifact 形状做实，再把发布渠道留给后续 milestone。
