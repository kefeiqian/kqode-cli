U9 的 packaged 测试重点在 [`materializeBackend.test.ts`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/tui/src/backend/packaged/__tests__/materializeBackend.test.ts)。它验证不同版本、不同 SHA-256、cache 复用、digest mismatch、权限、symlink、并发写入和失败路径。

```ts
it('isolates different backend contents at the same version by hash', async () => {
  const cacheBaseDir = tempBase();
  const oldBytes = Buffer.from('backend build A');
  const newBytes = Buffer.from('backend build B');
  const a = await materializePackagedBackend({ asset: asset(oldBytes), version: '0.1.0', cacheBaseDir });
  const b = await materializePackagedBackend({ asset: asset(newBytes), version: '0.1.0', cacheBaseDir });

  expect(path.dirname(a)).not.toBe(path.dirname(b));
  expect(fs.readFileSync(a)).toEqual(oldBytes);
  expect(fs.readFileSync(b)).toEqual(newBytes);
});
```

![U9 测试覆盖内容寻址和 packaged launch](../../images/u9-standalone-executable/u9-packaged-tests.png)

为什么测试这么细？因为 materialize 是 standalone executable 的信任边界。第一次运行就要写一个可执行文件；如果 symlink、宽松权限、digest mismatch 或 Windows 文件锁处理不好，后续发布渠道越多，问题越难查。

Cargo-facing 入口在 [`xtask/src/commands/package.rs`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/xtask/src/commands/package.rs)：

```rust
pub fn run(repo_root: &Path) -> Result<(), String> {
    cargo::build_release_bin(repo_root, BACKEND_BIN)?;
    bun::ensure_tui_dependencies(repo_root)?;
    bun::run(repo_root, &["run", PACKAGE_SCRIPT])
}
```

[`xtask/src/commands/tui/prod.rs`](https://github.com/kefeiqian/KQode/blob/f642fb3b65bad29da3497d77967af741424205ca/xtask/src/commands/tui/prod.rs) 则构建后直接运行 packaged executable：

```rust
pub fn run(repo_root: &Path) -> Result<(), String> {
    dev::ensure_workspace(repo_root)?;
    package::run(repo_root)?;

    let exe = paths::tui_packaged_exe(repo_root);
    if !exe.is_file() {
        return Err(format!(
            "packaged executable not found at {}; packaging did not produce it",
            exe.display()
        ));
    }
```

这样 contributor 仍然使用 `cargo xtask package` 和 `cargo xtask tui-prod`，而不是手动组合 Cargo、Bun、路径和 fixture。U9 还新增 `scripts/xtask.ps1` / `scripts/xtask.sh`，避免 Windows 上并行或长运行 `cargo xtask` 锁住同一个 `target\debug\xtask.exe`。
