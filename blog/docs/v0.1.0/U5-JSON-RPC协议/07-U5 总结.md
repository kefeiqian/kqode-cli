title: 7. U5 总结
---

U5 把 TypeScript TUI 侧的 JSON-RPC protocol contract 定义出来了。它交付的是 method name、notification name、params/result 类型、backend client seam、错误分类、`vscode-jsonrpc` descriptor 和对应测试。

## 关键技术决策

- 把 wire contract 放在 `@contracts/backend`。WHY：UI/state 和 backend 实现都需要这些类型，dependency-free seam 可以避免 layer cycle。
- 使用命名常量表达 method、ACK、notification 和 error kind。WHY：这些值跨 Rust / TypeScript，散落字符串容易漂移。
- 用 `RequestType` / `NotificationType0` 接入 `vscode-jsonrpc`。WHY：KQode 拥有协议语义，成熟库负责 JSON-RPC mechanics。
- 定义窄 `BackendClient` seam。WHY：UI 不应该依赖 process launcher 或 connection 生命周期。
- 预先区分 `protocol`、`transport`、`timeout`、`launch`。WHY：不同失败来源需要不同提示和恢复策略。

## 刻意延后内容

- source backend process launcher。
- 完整 process JSON-RPC backend client 生命周期。
- App submit 状态、消息队列、ACK 展示和错误展示。
- Rust 侧 `kqode.backend.ready` notification 实现。
- session、model、workspace、attachments 和 tool context 等更大 prompt payload。
