U5 在 [`tui/package.json`](https://github.com/kefeiqian/KQode/blob/ac566c70f9a1fdd1bfcb20e37b4d648f8402992f/tui/package.json) 中新增了 `vscode-jsonrpc` 和 `tree-kill`。这两个依赖分别服务不同阶段：前者立刻用于 protocol descriptor 和 connection 测试，后者为后续 backend process lifecycle 做准备。

```json
"dependencies": {
  "ink": "^7.1.0",
  "jotai": "^2.20.1",
  "react": "^19.2.7",
  "tree-kill": "^1.2.2",
  "vscode-jsonrpc": "^9.0.0"
}
```

选择 `vscode-jsonrpc` 的 WHY 和 U4 选择 `lsp-server` 类似：KQode 应该定义自己的 method 和 wire shape，但不应该重写 request ID、response promise、stream reader/writer 和 JSON-RPC error propagation。成熟库负责 transport mechanics，KQode 的代码负责产品协议。

`tree-kill` 在 U5 的 protocol 文件中还没有直接出现，因此不能把它描述成已完成的 process cleanup 实现。能从这个 commit 确认的是：TUI package 开始为后续跨平台 backend child process 清理准备依赖，尤其是 Windows 上只杀父进程可能留下子进程的问题。

U5 还在 manifest 里新增了 packaging scripts：

```json
"scripts": {
  "dev": "tsx main.tsx",
  "typecheck": "tsc --noEmit",
  "test": "vitest run",
  "package": "bun scripts/buildPackaged.ts",
  "package-release": "bun scripts/packageRelease.ts"
}
```

这些 script 只是 manifest 里的入口，U5 本身没有新增对应实现文件。因此这里更准确的说法是“预留 packaged artifact 入口”，而不是“已经完成打包发布”。

## JSON-RPC error 映射成 typed client error

[`messageProtocol.test.ts`](https://github.com/kefeiqian/KQode/blob/ac566c70f9a1fdd1bfcb20e37b4d648f8402992f/tui/src/backend/protocol/__tests__/messageProtocol.test.ts) 还验证了错误映射：

```ts
it('surfaces JSON-RPC method errors as typed protocol client errors', async () => {
  const { client, server } = pairedConnections();
  server.onRequest(messageSubmitRequest, () => {
    throw new ResponseError(ErrorCodes.InvalidParams, 'invalid message submit params');
  });

  const submit = createMessageConnectionClient(client).submitMessage({ text: 'boom' });

  await expect(submit).rejects.toBeInstanceOf(BackendClientError);
  await expect(submit).rejects.toMatchObject({ kind: BackendErrorKind.Protocol });
});
```

这条测试的价值在于限制错误泄漏方式。UI 不应该直接看到裸 `ResponseError`，因为那会把 transport library 的细节带进 render/state 层。U5 希望上层只处理 `BackendClientError`，并通过 `BackendErrorKind.Protocol` 判断这是前后端 protocol contract 问题。

这个设计也和 U4 的错误语义对应：Rust 端把 invalid params 表达为 JSON-RPC `-32602`；TypeScript 端把这类 response error 收敛为 `protocol` kind。后续 UI 可以基于 kind 做更清楚的提示。
