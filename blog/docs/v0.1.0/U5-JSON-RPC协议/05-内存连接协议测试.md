title: 5. 内存连接协议测试
---

U5 的测试没有启动真实 Rust backend，因为这个 commit 的职责是 TypeScript 协议定义，不是 process integration。它用内存 stream 创建一对 `vscode-jsonrpc` connection，验证 descriptor、client seam 和 ACK shape 可以在真实 JSON-RPC connection 上工作。

[`tui/src/backend/protocol/__tests__/messageProtocol.test.ts`](https://github.com/kefeiqian/KQode/blob/ac566c70f9a1fdd1bfcb20e37b4d648f8402992f/tui/src/backend/protocol/__tests__/messageProtocol.test.ts) 里的连接 helper 如下：

```ts
function pairedConnections(): PairedConnections {
  const clientToServer = new PassThrough();
  const serverToClient = new PassThrough();
  const client = createMessageConnection(
    new StreamMessageReader(serverToClient),
    new StreamMessageWriter(clientToServer)
  );
  const server = createMessageConnection(
    new StreamMessageReader(clientToServer),
    new StreamMessageWriter(serverToClient)
  );
  client.listen();
  server.listen();

  const pair: PairedConnections = {
    client,
    server,
    dispose: () => {
      client.dispose();
      server.dispose();
    }
  };
  openPairs.push(pair);
  return pair;
}
```

这种测试形态的好处是轻量且真实：它不依赖 OS child process，但仍然走 `StreamMessageReader`、`StreamMessageWriter` 和 `MessageConnection`。换句话说，它验证的是 TypeScript protocol 层，而不是 mock 一个普通函数调用。

## Fake server 使用同一个 descriptor

测试 server 注册的不是裸字符串，而是 U5 新增的 `messageSubmitRequest`：

```ts
function ackingServer(server: MessageConnection): void {
  server.onRequest(messageSubmitRequest, ({ text }) => ({
    message: ACK_MESSAGE,
    receivedText: text
  }));
}
```

这很重要。如果测试端也手写 `'kqode.message.submit'`，就无法证明 descriptor 是真正的 single source of truth。让 fake server 和 client 都通过 `messageSubmitRequest` 通信，能固定 U5 想要的使用方式：call site 不写 method string。

## ACK 和文本保真

第一条测试覆盖基本 ACK：

```ts
it('sends kqode.message.submit and receives an ACK success response', async () => {
  const { client, server } = pairedConnections();
  ackingServer(server);

  const result = await createMessageConnectionClient(client).submitMessage({
    text: 'hello from tui'
  });

  expect(result.message).toBe(ACK_MESSAGE);
  expect(result.receivedText).toBe('hello from tui');
});
```

第二条测试验证 Unicode、空格和换行保真：

```ts
it('preserves Unicode, surrounding spaces, and newlines in receivedText', async () => {
  const { client, server } = pairedConnections();
  ackingServer(server);

  const text = '  café\n☕ 日本語  ';
  const result = await createMessageConnectionClient(client).submitMessage({ text });

  expect(result.receivedText).toBe(text);
});
```

这和 U4 的 Rust 测试形成对照：Rust 端证明 stdio frame 不改写文本，TypeScript 端证明 message connection client 也不改写文本。
