U12 对应 commit [`a667e95e`](https://github.com/kefeiqian/KQode/commit/a667e95e4ba1832be0b94fb4fb027cae2f5438ad)，主题是 `feat(release): prepare cross-platform distribution artifacts`。它把前面能在本地运行的 KQode，推进到“可以被分发”的阶段：为当前 host 打包 standalone executable，定义跨平台 artifact 命名，增加 npm thin launcher，并写下 Homebrew / winget / GitHub Release 的注册说明。

![U12 跨平台分发链路](../../images/u12-cross-platform-distribution/distribution-flow.png)

## 这个提交实际新增了什么

U12 的核心新增文件包括：

- [`tui/scripts/packageRelease.ts`](https://github.com/kefeiqian/KQode/blob/a667e95e4ba1832be0b94fb4fb027cae2f5438ad/tui/scripts/packageRelease.ts)：把 standalone `kqode` executable 归档成 release asset 并生成 checksum。
- [`xtask/src/commands/package_release.rs`](https://github.com/kefeiqian/KQode/blob/a667e95e4ba1832be0b94fb4fb027cae2f5438ad/xtask/src/commands/package_release.rs)：提供 Cargo-facing `package-release` wrapper。
- [`packaging/npm/kqode/package.json`](https://github.com/kefeiqian/KQode/blob/a667e95e4ba1832be0b94fb4fb027cae2f5438ad/packaging/npm/kqode/package.json)、[`bin/kqode.cjs`](https://github.com/kefeiqian/KQode/blob/a667e95e4ba1832be0b94fb4fb027cae2f5438ad/packaging/npm/kqode/bin/kqode.cjs)、[`lib/install.cjs`](https://github.com/kefeiqian/KQode/blob/a667e95e4ba1832be0b94fb4fb027cae2f5438ad/packaging/npm/kqode/lib/install.cjs)、[`lib/resolve.cjs`](https://github.com/kefeiqian/KQode/blob/a667e95e4ba1832be0b94fb4fb027cae2f5438ad/packaging/npm/kqode/lib/resolve.cjs)：npm 分发包。
- [`docs/release/kqode_distribution_registration.md`](https://github.com/kefeiqian/KQode/blob/a667e95e4ba1832be0b94fb4fb027cae2f5438ad/docs/release/kqode_distribution_registration.md)：发布通道注册指南。

`xtask` wrapper 很薄：

```rust
pub fn run(repo_root: &Path) -> Result<(), String> {
    package::run(repo_root)?;
    bun::run(repo_root, &["run", RELEASE_SCRIPT])
}
```

这句代码表达了 U12 的架构边界：Rust `xtask` 负责编排，已有 `package` 命令负责构建 standalone executable，TypeScript script 负责 archive / checksum 这类和 npm 分发也共享的逻辑。

## 为什么先做“当前 host”打包

U12 没有在本地脚本里硬做 cross compile。它的 `packageRelease.ts` 明确写着：每个 CI runner 为自己的 target 运行一次，本地命令也只产出当前 host 的 asset。这样设计是因为 KQode 的最终产物包含 Bun standalone executable 和 Rust backend，跨平台打包最稳妥的方式是 native runner 构建，而不是在一台机器上模拟所有平台。

这带来一个很重要的拆分：U12 定义产物格式与本地命令；U13 再用 GitHub Actions matrix 把这些命令跑到多个 native runner 上。

## 为什么分发文档也属于这个提交

分发不是单一技术动作，而是一组渠道协作：GitHub Release 提供原始 artifacts，npm 负责下载并安装对应 artifact，Homebrew 和 winget 则引用同一组 GitHub Release URL 和 checksum。如果这些约定不在代码提交里沉淀，后续 CI 很容易“能跑但不知道为什么这样跑”。
