U12 的 [`packageRelease.ts`](https://github.com/kefeiqian/KQode/blob/a667e95e4ba1832be0b94fb4fb027cae2f5438ad/tui/scripts/packageRelease.ts) 负责把已经构建好的 standalone executable 包成 release asset。它不是编译脚本，而是打包脚本；编译由 `cargo xtask package` 完成，打包由 `package-release` 接着做。

```ts
/**
 * Packages the current host's standalone `kqode` executable into a
 * direct-download release archive plus checksums under `tui/dist/release/`.
 *
 * Produces `kqode-<target>.tar.gz` (POSIX) or `kqode-<target>.zip` (Windows)
 * containing only the executable, a per-archive `kqode-<target>.sha256`, and an
 * aggregate `checksums.txt`.
 */
```

## 为什么 archive 里只放一个 executable

`createArchive` 会先把 executable 复制到 staging directory，并保证 archive 根目录只有一个文件：`kqode` 或 `kqode.exe`。

```ts
const staged = path.join(stageDir, host.binaryName);
fs.copyFileSync(exePath, staged);
if (process.platform !== 'win32') {
  fs.chmodSync(staged, 0o755);
}

const archivePath = path.join(releaseDir, host.archive);
const tarArgs =
  host.ext === 'zip'
    ? ['-a', '-cf', archivePath, '-C', stageDir, host.binaryName]
    : ['-czf', archivePath, '-C', stageDir, host.binaryName];
```

这样做让每个渠道的安装逻辑都很简单：下载 archive，校验 checksum，解压根目录下的 binary。它避免了不同平台 archive 内部目录层级不一致，也避免 npm / Homebrew / winget 各自写复杂路径规则。

## 为什么 Windows 是 `.zip`，其他平台是 `.tar.gz`

U12 在 host target 里直接决定扩展名：

```ts
const ext = platform === 'win32' ? 'zip' : 'tar.gz';
return { target, archive: `${target}.${ext}`, ext, binaryName: resolve.binaryName(platform) };
```

这是为了贴近平台习惯和下游工具。Windows 用户和 winget 更自然地消费 `.zip`；POSIX 平台使用 `.tar.gz` 更利于保留 executable bit。脚本在 POSIX staging 时显式 `chmod 0o755`，保证归档后仍然可执行。

## 为什么每个 target 有独立 `.sha256`

U12 为当前 host 产出两种 checksum 文件：

```ts
const digest = sha256File(archivePath);
const checksumLine = `${digest}  ${host.archive}\n`;
fs.writeFileSync(path.join(releaseDir, `${host.target}.sha256`), checksumLine);
fs.writeFileSync(path.join(releaseDir, 'checksums.txt'), checksumLine);
```

单 target `.sha256` 方便 CI 在 build job 上传并在 npm installer 下载某个目标的校验值；aggregate `checksums.txt` 则方便 GitHub Release 页面给人类和 Homebrew / winget maintainer 查看所有 target 的 digest。本地一次只能构建一个 host，所以本地 `checksums.txt` 只有一行；U13 的 release job 会把每个 `.sha256` 排序合并成完整 manifest。
