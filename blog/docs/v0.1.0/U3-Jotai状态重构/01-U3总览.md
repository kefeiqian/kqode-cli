[U2](/category/u2-interactive-home-screen) 用最直白的方式把交互式主界面跑通了：数据从 `main` 一路 props 透传，`HomeScreen` 和 `PromptComposer` 都是单文件巨石组件，状态用 `useState`/`useReducer`。它能跑、也好读，但也积累了三处明显的“重构信号”。U3 就是一次专门的架构重构——**行为基本不变，但把结构做扎实**。

对应 commit：[`5432e018`（refactor(tui): move home screen state into Jotai）](https://github.com/kefeiqian/KQode/commit/5432e018cb496e5f7359e69d47a2a7d1691c0794)。本篇及本系列 U3 的源码都 pin 到这个 commit。

## U2 留下的三处重构信号

- **props 透传开始爆炸**。主界面有十几个需要被多个组件读取的派生值：布局行数、滚动偏移、已提交消息、composer 行数……在 U2 里它们要么在 `HomeScreen` 里 `useState`、再一层层往下传，要么靠回调往上抬。叶子组件的参数列表越来越长。
- **巨石文件逼近 200 行红线**。`HomeScreen.tsx` 近 190 行、`PromptComposer.tsx` 约 300 行，已经超过 `tui/AGENTS.md`“文件保持在 200 行以内”的约定。
- **纯逻辑和展示混在一起**。`bodyRows.ts`、`layout.ts` 这些纯函数 helper 和 `.tsx` 组件一起放在 `components/` 下，边界不清。

## U3 做的四件事

1. **把共享状态迁移到 Jotai**。用一个独立的 Jotai store 承载主界面配置与所有派生状态，组件按需订阅自己关心的那一个 atom，不再靠 props 链。见[第 2 篇](./02-迁移到Jotai.md)。
2. **按 200 行约定拆分巨石组件**。`HomeScreen` 拆成 `HomeScreen/` 文件夹，`PromptComposer` 拆成 6 个聚焦模块。见[第 3 篇](./03-按200行拆分组件.md)。
3. **把纯 helper 抽到 `libs/tui`**。`bodyRows`、`layout`、`cwdLine`、背景块 glyph 都移到 `src/libs/tui/` 下，`components/` 只留 `.tsx`。见[第 4 篇](./04-纯helper抽到libs-tui.md)。
4. **切换到 Gemini 风格配色，并把 TUI 约定写进 `tui/AGENTS.md`**。见[第 5 篇](./05-Gemini配色与TUI约定.md)。

:::info 本篇不重复讲功能
U2 每个功能（布局、正文、输入框、Git、鼠标）“做了什么、为什么”已经在 [U2 系列](/category/u2-interactive-home-screen)里逐函数讲过。U3 是**结构重构**，所以这一系列只讲“为什么这样改、改完长什么样”，功能行为请对照 U2。
:::

## 数据流的变化

U2 里 `App` 直接把一堆 props 透传给 `HomeScreen`。U3 之后，`App` 先把外部 props 与终端尺寸合并成一份**完整配置** `HomeScreenConfig`，再交给 `HomeScreen`：

```tsx
export function App({ screen }: AppProps) {
  const windowSize = useWindowSize();
  const config = createHomeScreenConfig({
    ...screen,
    columns: screen.columns ?? windowSize.columns ?? DEFAULT_COLUMNS,
    rows: Math.max(MIN_ROWS, screen.rows ?? windowSize.rows ?? DEFAULT_ROWS)
  });

  return <HomeScreen config={config} />;
}
```

而 `HomeScreen` 不再是巨石组件，它变成**状态边界**：创建一个独立的 Jotai store、把 config 写进根 atom、再用 `Provider` 包住真正渲染的 `HomeScreenView`（第 2 篇细讲）。

## 组件树与状态边界

```text
App                        （合并 props + 尺寸 → config）
└─ HomeScreen              （创建 Jotai store，注入 config）
   └─ HomeScreenView       （读 layout 原子，处理滚动按键）
      ├─ Header
      ├─ HomeBody          → BodyPane
      ├─ HomeCwd           → CwdLine
      ├─ HomeComposer      → PromptComposer
      └─ HomeStatus        → StatusBar
```

`HomeScreenView` 里每一块都是一个只读若干 atom 的小包装组件（`HomeHeader`、`HomeBody`、`HomeCwd`、`HomeComposer`、`HomeStatus`），把 atom 值转成叶子组件的 props。叶子组件保持“纯展示 + 输入接线”，不直接耦合全局状态。
