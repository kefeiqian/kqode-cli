U8 新增 [`tui/src/state/backend/bodyEntries.ts`](https://github.com/kefeiqian/KQode/blob/bcc8130e137f37b8dc0884918768f3b1fc5fc1ef/tui/src/state/backend/bodyEntries.ts)，把 queue item 转成 body 可以渲染的 entry。

```ts
export function queueToBodyEntries(queue: readonly QueueItem[]): BodyEntry[] {
  return queue.flatMap((item) => {
    const promptText = sanitizeDisplayText(item.text);
    const promptEntry: BodyEntry =
      item.state === 'queued'
        ? { id: `prompt-${item.id}`, kind: 'pending', text: promptText }
        : { id: `prompt-${item.id}`, kind: 'prompt', text: promptText };

    return item.result === undefined
      ? [promptEntry]
      : [promptEntry, { id: `result-${item.id}`, kind: item.result.kind, text: item.result.text }];
  });
}
```

![正文显示 prompt、pending、ACK 和错误](../../images/u8-submit-ack-wiring/body-entry-states.png)

这里没有把正文存成普通字符串，而是保留 `prompt`、`pending`、`success`、`error` 等 kind。原因是 transcript 后面会承载更多语义：assistant message、tool output、审批提示、错误恢复都需要不同样式和行为。现在就保留结构，后续不用反向解析字符串。

显示文本统一走 [`sanitizeDisplayText`](https://github.com/kefeiqian/KQode/blob/bcc8130e137f37b8dc0884918768f3b1fc5fc1ef/tui/src/libs/text/sanitizeDisplayText.ts)：

```ts
export const DISPLAY_OUTPUT_MAX_BYTES = 128 * 1024;

const CONTROL_CHAR_PATTERN = /[\u0000-\u0008\u000b-\u001f\u007f-\u009f]/g;

export function sanitizeDisplayText(text: string, maxBytes = DISPLAY_OUTPUT_MAX_BYTES): string {
  return capUtf8Bytes(text, maxBytes).replace(CONTROL_CHAR_PATTERN, escapeControlChar);
}
```

为什么必须净化？因为 prompt 和 ACK 内容最终都来自用户输入；后端错误以后也可能携带外部文本。如果里面有 ESC、ANSI CSI、OSC、回车或退格，terminal 可能执行清屏、移动光标或伪造 UI。U8 不静默删除，而是显示成可见的 `\xNN`，这样既安全，也方便排查。

错误同样进入正文模型：

```ts
export function backendErrorMessage(error: unknown): string {
  if (error instanceof BackendClientError) {
    return `Rust backend failed: ${error.message}`;
  }
  return `Rust backend failed: ${error instanceof Error ? error.message : String(error)}`;
}
```

错误不是 console log，而是和 prompt 配对的 result entry。用户能看到哪条 prompt 失败，测试也能按 transcript 验证失败路径。
