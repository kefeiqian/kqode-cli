U8 的主测试是 [`tui/src/__tests__/App.submit.test.tsx`](https://github.com/kefeiqian/KQode/blob/bcc8130e137f37b8dc0884918768f3b1fc5fc1ef/tui/src/__tests__/App.submit.test.tsx)。它用 fake backend client 验证 UI 接线，而不是重复启动真实 Rust 进程。

```ts
it('appends the prompt and the Rust backend ACK when Enter is pressed', async () => {
  const submitMessage = echoBackend();
  const { lastFrame, stdin } = renderApp({ submitMessage });

  await submit(stdin, 'hello from tui');

  const frame = await waitForFrame(lastFrame, (output) =>
    output.includes('Rust backend ACK - received: hello from tui')
  );
  expect(frame).toContain('❯ hello from tui');
  expect(submitMessage).toHaveBeenCalledWith({ text: 'hello from tui' });
});
```

![U8 测试验证 prompt 和 ACK 同屏出现](../../images/u8-submit-ack-wiring/app-submit-test.png)

为什么用 fake client？因为 U7 已经覆盖过 backend client 和 JSON-RPC 进程路径；U8 要证明的是 Enter 是否调用 client、原始文本是否保留、返回值是否进入 frame。这样测试失败时，定位范围更小。

连续提交测试覆盖 FIFO 中间状态：

```ts
await submit(stdin, 'first');
await submit(stdin, 'second');
await submit(stdin, 'third');

const queuedFrame = await waitForFrame(
  lastFrame,
  (output) => output.includes('second (pending)') && output.includes('third (pending)')
);
expect(submitMessage).toHaveBeenCalledTimes(1);
expect(submitMessage).toHaveBeenCalledWith({ text: 'first' });
```

这验证第二、第三条 prompt 会立即显示，但不会抢在第一条之前发送。

安全测试在 [`sanitizeDisplayText.test.ts`](https://github.com/kefeiqian/KQode/blob/bcc8130e137f37b8dc0884918768f3b1fc5fc1ef/tui/src/libs/text/__tests__/sanitizeDisplayText.test.ts)：

```ts
it('escapes ESC, ANSI CSI, OSC, carriage returns, and backspaces', () => {
  const sanitized = sanitizeDisplayText('evil\u001b[2J\rrow\bback\u0007bell');

  expect(sanitized).toBe('evil\\x1b[2J\\x0drow\\x08back\\x07bell');
  expect(sanitized).not.toContain('\u001b');
  expect(sanitized).not.toContain('\r');
});
```

[`backendIsolation.test.ts`](https://github.com/kefeiqian/KQode/blob/bcc8130e137f37b8dc0884918768f3b1fc5fc1ef/tui/src/__tests__/backendIsolation.test.ts) 还扫描 `components` 和 `state`，禁止它们引用 `node:child_process`、`launchSourceBackend`、`tree-kill` 等 process / launch 细节。这是在保护架构：UI 和 state 层只能通过注入的 backend client seam 触达后端。
