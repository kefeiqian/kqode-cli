title: 7. U7 总结
---

U7 把 U6 启动出来的 Rust 后端变成了 UI 可用的 JSON-RPC client。到这个里程碑，KQode TUI 已经不只是能启动一个后端进程，而是可以等待协议级 ready、发送 `kqode.message.submit` request、收到 ACK response、处理错误、在 fatal failure 后 dead，并在下一次 submit 时重启。

## 交付物

- Rust backend 在 stdio transport 建立后发送 `kqode.backend.ready` notification。
- TypeScript 侧用 `vscode-jsonrpc` 创建 `MessageConnection`，通过 ready notification 判断 startup readiness。
- `createMessageConnectionClient` 把 JSON-RPC connection 包成窄 `BackendClient`。
- `createBackendClient` 管理 `idle` / `starting` / `ready` / `closing` / `dead` lifecycle。
- `startBackendRuntime` 把 client 注入 UI store，显示 startup hint，并在启动失败后保留 retry seam。
- Rust、TypeScript client、runtime store 三层测试覆盖 happy path 和关键错误路径。

## 关键决策与原因

### 1. ready 是协议级 notification

```rust
connection
    .sender
    .send(Message::Notification(Notification::new(
        BACKEND_READY_METHOD.to_owned(),
        (),
    )))
```

**为什么：** OS spawn 只能说明进程存在，不能说明 JSON-RPC transport 可用。用 notification 表示 ready，可以避免把第一个用户 request 当成探活请求，也可以防止“进程启动但不说协议”的假成功。

### 2. request / response correlation 交给 `vscode-jsonrpc`

**为什么：** request id 匹配、pending map、response dispatch 都是成熟库已经解决的问题。KQode 自己更应该关注 lifecycle、错误分类和 UI seam，而不是手写一套容易出错的 JSON-RPC dispatcher。

### 3. protocol error 可恢复，timeout / transport error fatal

**为什么：** method-level `ResponseError` 只说明某个请求被拒绝，不代表连接坏了；timeout 和 transport failure 则意味着当前 connection 的有序性和可用性不可靠。区分二者后，client 可以继续使用健康连接，也能在 fatal 情况下主动 teardown。

### 4. eager start 失败后保留 dead-state client

```ts
.catch(() => {
  // Keep the Dead-state client in the seam (do not dispose or clear it): the
  // next submit retries through ensureSession() and, on repeat failure,
  // settles a visible backend-error entry rather than silently dropping it.
})
```

**为什么：** 如果启动失败就清空 client seam，后续 prompt 可能被 UI 路径静默丢掉。保留 dead-state client 让下一次 submit 可以重试；如果仍失败，也会落成可见 error entry。

## 暂缓事项

- U7 只实现 `message.submit`，还没有 session create / restore、tool calls、streaming events 或 MCP 方法。
- U7 不做自动重放 interrupted request，避免重复执行用户没有确认的操作。
- U7 没有实现 packaged backend client；仍然基于 U6 source-mode launcher。
- U7 没有把 stderr diagnostics 变成 UI 面板；错误分类已经准备好，展示层后续再接。
- U7 没有实现 daemon mode；当前仍是 TUI 拥有一个 child backend process。
