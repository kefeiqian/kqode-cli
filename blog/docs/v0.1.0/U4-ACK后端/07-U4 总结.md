title: 7. U4 总结
---

U4 把 KQode 从 starter Rust binary 推进到第一个可被前端驱动的 Rust JSON-RPC backend。它交付的是隐藏 backend 进程、stdio JSON-RPC loop、`kqode.message.submit` ACK 响应，以及覆盖成功和错误路径的进程级测试。

## 关键技术决策

- 使用隐藏 backend 参数。WHY：这是 TUI 内部 transport，不是用户稳定 CLI surface。
- 使用 `lsp-server` 承担 JSON-RPC stdio framing。WHY：当前要验证进程协议链路，不要先重写 transport parser。
- 集中 protocol 常量。WHY：method、ACK 文案和 error code 跨 Rust / TypeScript，散落字符串容易漂移。
- 对 params 使用 `deny_unknown_fields`。WHY：协议演进必须显式同步，不能静默接受前端多发字段。
- 用进程级测试。WHY：真实风险在 binary invocation、stdio frame 和退出语义，不在纯函数本身。

## 刻意延后内容

- `kqode.backend.ready` notification。
- TypeScript process launcher 和 JSON-RPC client。
- App submit 状态与 ACK 展示。
- agent loop、provider、tool、session store。
- 公开用户 CLI 产品形态。
