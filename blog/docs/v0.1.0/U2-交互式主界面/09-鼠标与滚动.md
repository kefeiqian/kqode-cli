正文能滚动，靠的是鼠标滚轮和键盘（PageUp/PageDown/End）。U2 还没有做滚动条或鼠标点击，先把“让正文往上或往下挪几行”跑通。

![KQode 主界面输入区和底部状态栏高亮](../../images/mouse-scroll/input-area-highlight.png)

滚轮这一块容易误解：终端不会给 Ink 一个浏览器式的 `WheelEvent`。开启鼠标追踪后，终端会把鼠标事件编码成一串转义序列写进 `stdin`，程序再自己解析。相关代码在 [`tui/src/libs/terminal/mouse.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/libs/terminal/mouse.ts) 和 [`tui/src/components/HomeScreen.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/HomeScreen.tsx) 的滚动接线里。

## SGR 鼠标追踪是什么

默认情况下，终端不会把滚轮或点击上报给程序。要收到鼠标事件，程序得先写控制序列，请终端开启 SGR 鼠标追踪；退出时再写另一组序列关掉：

```ts
export const ENABLE_SGR_MOUSE_TRACKING = '\u001B[?1000h\u001B[?1006h';
export const DISABLE_SGR_MOUSE_TRACKING = '\u001B[?1006l\u001B[?1000l';
```

可以拆开看：

- `\u001B[` 是控制序列开头，也就是 ESC + `[`。
- `?1000h` 开启鼠标按钮上报，终端开始把点击和滚轮写回程序。
- `?1006h` 切到 SGR 扩展鼠标编码，事件变成 `ESC [ < button ; x ; y M` 这种文本格式，比老格式更适合解析。
- 关闭时用 `l` 复位：先关 `1006`，再关 `1000`。

这个组件在挂载时开启、卸载时关闭，而且只在 TTY 下操作：

```tsx
useEffect(() => {
  if (!stdout.isTTY) {
    return;
  }
  stdout.write(ENABLE_SGR_MOUSE_TRACKING);
  return () => {
    stdout.write(DISABLE_SGR_MOUSE_TRACKING);
  };
}, [stdout]);
```

这里的 TTY（[Teletypewriter](https://en.wikipedia.org/wiki/Teleprinter)）已经不是字面上的电传打字机，而是操作系统里“连着交互式终端”的抽象。`stdout.isTTY === true` 表示输出面向一个终端窗口；如果输出被管道、CI、测试框架或 `kqode > out.txt` 这类重定向接走，通常就是非 TTY。

非 TTY 下写这些序列没有意义，还会污染日志、快照或重定向文件。清理函数同样重要：程序退出时必须把鼠标追踪关掉，不然终端可能继续把鼠标事件当作程序输入。

## 解析滚轮方向

开启 `1006` 后，上滚可能变成 `\u001B[<64;12;7M`。这个序列可以按 `ESC [ < buttonCode ; column ; row M` 理解：`buttonCode` 记录按钮类型和修饰键，`column` / `row` 是终端坐标，末尾的 `M` 表示按下或滚动事件。[`tui/src/libs/terminal/mouse.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/libs/terminal/mouse.ts) 用一个正则把它拆开，判断是不是滚轮、往哪个方向：

```ts
const SGR_MOUSE_INPUT_PATTERN = /^(?:\u001B)?\[<(?<buttonCode>\d+);\d+;\d+(?<eventType>[mM])$/;
const WHEEL_BUTTON_OFFSET = 64;
const WHEEL_BUTTON_COUNT = 4;

export function parseMouseWheelInput(input: string): MouseWheelDirection | null {
  const match = SGR_MOUSE_INPUT_PATTERN.exec(input);
  if (match?.groups === undefined || match.groups.eventType !== 'M') {
    return null;
  }
  const buttonCode = Number.parseInt(match.groups.buttonCode, 10);
  if (buttonCode < WHEEL_BUTTON_OFFSET) {
    return null;
  }
  const wheelButton = (buttonCode - WHEEL_BUTTON_OFFSET) % WHEEL_BUTTON_COUNT;
  if (wheelButton === 0) return 'up';
  if (wheelButton === 1) return 'down';
  return null;
}
```

关键是 `buttonCode`。SGR 编码里，滚轮事件从 `64` 起（`WHEEL_BUTTON_OFFSET`）：`64` 是上滚，`65` 是下滚。先减去偏移，是为了把“滚轮组的起点”归零：

- `64 - 64 = 0`，上滚。
- `65 - 64 = 1`，下滚。
- `66 - 64 = 2` 和 `67 - 64 = 3` 是另外两个滚轮槽位，U2 暂时不处理。

再 `% 4`，是因为鼠标协议会把 Shift、Alt、Ctrl 这类修饰键也塞进同一个 `buttonCode`。修饰键不应该改变滚轮方向；取模后只保留 `0..3` 这个方向槽位，所以带修饰键的上滚仍然会落到 `0`。

举个例子：普通上滚是 `64`，Shift + 上滚常见编码会变成 `68`。两者减去偏移后分别是 `0` 和 `4`，再 `% 4` 都得到 `0`，所以都解析成 `'up'`。普通下滚是 `65`，Alt + 下滚可能变成 `73`；`65 - 64 = 1`，`73 - 64 = 9`，`9 % 4 = 1`，方向仍然是 `'down'`。这就是“先减偏移、再取模”的意义：先进入滚轮自己的编号空间，再把修饰键带来的高位信息剥掉。最后代码只接受 `0` 和 `1`，其余滚轮槽位返回 `null`。

非滚轮的鼠标事件会被挡掉：普通点击的 `buttonCode` 小于 `64`，释放事件的末尾是 `m` 而不是 `M`。U2 只关心滚动，点击暂时不处理。还有一个 `isMouseInput` 只判断“是不是鼠标序列”，[`tui/src/components/PromptComposer.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/PromptComposer.tsx) 用它来跳过这些序列（[第 7 篇](./07-输入框渲染与输入.md)），避免鼠标乱码被当成文本输入。

## 把方向接到滚动

`HomeScreen` 的 `useInput` 把滚轮方向和键盘键统一汇聚到 `scrollBodyByRows`：

```tsx
useInput((input, key) => {
  const wheelDirection = parseMouseWheelInput(input);
  if (wheelDirection !== null) {
    scrollBodyByRows(wheelDirection === 'up' ? MOUSE_WHEEL_SCROLL_ROWS : -MOUSE_WHEEL_SCROLL_ROWS);
    return;
  }
  if (key.pageUp) { scrollBodyByRows(bodyScrollPageRows); return; }
  if (key.pageDown) { scrollBodyByRows(-bodyScrollPageRows); return; }
  if (key.end) { setBodyScrollOffsetRows(0); }
});
```

设计细节：

- 滚轮一次滚 `MOUSE_WHEEL_SCROLL_ROWS = 3` 行，符合大多数终端的滚动手感。
- PageUp/PageDown 滚一“页”，页高 `bodyScrollPageRows = max(1, bodyRows - 2)`——留 2 行重叠，翻页时上下文不断。
- **End 直接归零**回到贴底（最新内容），这是“跳到最新”的快捷键。

而 `scrollBodyByRows` 本身负责把偏移处理在合法区间：

```tsx
const scrollBodyByRows = (deltaRows: number) => {
  setBodyScrollOffsetRows((current) => Math.min(maxBodyScrollOffsetRows, Math.max(0, current + deltaRows)));
};
```

`Math.max(0, ...)` 保证不会翻过最新内容，`Math.min(maxBodyScrollOffsetRows, ...)` 保证不会翻过最旧内容（`maxBodyScrollOffsetRows` 来自第 3、4 篇的正文行数与可见行数之差）。
