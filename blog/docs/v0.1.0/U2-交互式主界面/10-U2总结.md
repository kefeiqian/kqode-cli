前面九篇把 U2 的每个模块都逐函数拆了一遍。这一篇做个收束：U2 交付了什么、做了哪些关键技术决策、以及刻意留到后面的部分。

U2 对应主界面计划 [`2026-06-25-003`](https://github.com/kefeiqian/KQode/blob/ad9670d1978ae2c6e738ea76bc52b58149463dbd/docs/plans/2026-06-25-003-feat-first-ink-tui-homepage-plan.md) 的 U2（静态主界面组件）与 U3（输入框输入与换行），以及主题计划 [`2026-06-29-001`](https://github.com/kefeiqian/KQode/blob/ad9670d1978ae2c6e738ea76bc52b58149463dbd/docs/plans/2026-06-29-001-feat-gemini-style-tui-theming-plan.md) 的半行背景块，落在一个 commit [`dd15b678`](https://github.com/kefeiqian/KQode/commit/dd15b678392eacc2ffcee88884eba18ae52c1236) 里。

## 到目前为止完成了什么

U1 的 App 还只是一个静态空壳。U2 把它变成了一个真正能交互的主界面：

- **可组合的主界面布局**：Header、正文转录区、cwd 行、输入框、状态栏五块，由 [`HomeScreen`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/HomeScreen.tsx) 纵向组合并自持状态。
- **底部吸附布局**：cwd、输入框、状态栏在任意终端尺寸下都吸在底部，正文与 cwd 之间恰好一个空行，靠 `resolveHomeScreenLayout` 与 spacer 计算实现。
- **正文转录与滚动**：[`bodyRows`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/bodyRows.ts) 把类型化 item 编译成渲染行，[`BodyPane`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/BodyPane.tsx) 处理换行、滚动窗口和滚动条，支持滚轮与 PageUp/PageDown/End。
- **可输入可提交的输入框**：[`composerReducer`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/state/composerReducer.ts) 持有状态，[`PromptComposer`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/PromptComposer.tsx) 负责输入、换行、Unicode 光标、64 KiB 限长、提交校验。
- **Git 感知的 cwd**：[`gitStatus`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/libs/git/gitStatus.ts) 解析 porcelain 输出，cwd 行显示 `~` 相对路径加分支/改动标记。
- **内部主题与半行背景块**：[`themeConfig`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/theme/themeConfig.ts) 集中语义令牌（当前是 GitHub 深色风），[`BackgroundBlock`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/BackgroundBlock.tsx) 用半行 `▄`/`▀` glyph 给消息块和输入框做半行留白的背景。

## 关键技术决策

- **先用最直白的方式跑通**。这一版没有全局状态容器，数据从 `main` 一路 props 透传；`HomeScreen`、`PromptComposer` 都是单文件巨石组件，状态用 `useState`/`useReducer`。在只有一个屏、状态还不多时，这是成本最低、最好读的方案。等真出现“派生状态多、props 链长、文件逼近 200 行”的痛点，再重构——这正是 [U3](/category/u3-jotai-state-refactor) 要做的。

- **纯函数与展示分离，隔离易错逻辑**。布局（`resolveHomeScreenLayout`）、正文行编译（`bodyRows`）、Git 解析（`gitStatus`）、文本裁剪（`clipText`）都是可单测的纯函数，组件只负责“用算好的结果渲染”。

- **背景渲染是内部原语，且 fallback-aware**。半行块只用在消息块和输入框两个局部，没有去刷整屏（早期尝试会露行缝）。`shouldRenderBackgroundBlock` 会按 `NO_COLOR`、屏幕阅读器、色深降级。所有“含义”都用非颜色标记（`ERROR:`、`•`、`❯`、`*`/`+`/`%`、滚动条字形）承载，去色后依然可读。

- **外部依赖一律优雅降级**。Git 状态读取整段 `try/catch` 加 2 秒超时，失败就不显示而不是崩溃；SGR 鼠标追踪只在 TTY 下开关，非 TTY 跳过；退出时一定关闭鼠标追踪。

- **Unicode 正确性从一开始就做对**。光标移动和删除按 code point 边界（处理代理对），限长按 UTF-8 字节数算。这些“容易被忽略、后期难补”的正确性问题，在第一版就处理掉。

- **手动光标定位必须随布局同步验证**。光标 `y` 是 `composerTop` + 可见行号 + Ink 行原点偏移的叠加，任何改动 body 高度、spacer、换行或校验行的改动都要重新确认光标落在当前文本行上。

- **按宽度分档降级**。Header（36/52 列阈值）和 StatusBar（60 列阈值）都会随宽度收缩内容，且 `Header` 与 `headerRowCount` 共用同一组阈值常量，保证“显示什么”和“占几行”永远一致。

## 刻意留到后面的部分

U2 只搭了**可交互的前端外壳**，没有接任何后端行为。提交 prompt 目前只是把它追加进正文，`onPromptSubmit` 默认是 noop。主题也没有任何用户可见配置。这些都按计划留给后续单元：

- [U3](/category/u3-jotai-state-refactor)：把共享状态迁移到 Jotai、按 200 行约定拆分巨石组件、把纯 helper 收进 `libs/tui`，并切换到 Gemini 风格配色。
- [U4](/category/u4-ack-backend)–[U8](/category/u8-submit-ack-wiring)：Rust JSON-RPC 后端、协议、受控启动器、后端客户端、提交/ACK 接线。
- 之后的 [U9](/category/u9-standalone-executable)–[U13](/category/u13-release-pipeline)：独立可执行文件、架构加固、终端控制、跨平台分发与 Release 流水线。

换句话说，U2 先把主界面这层“看得见、摸得着”的部分做扎实：布局稳、输入顺、转录清楚，后面接入真正的 agent 循环时也就有了可靠的 UI 基础。
