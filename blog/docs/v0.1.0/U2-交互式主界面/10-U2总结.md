前面九篇介绍了 U2 的功能：主题、布局、正文转录、cwd、输入框、顶部状态栏、鼠标滚动都各自讲过。这一篇只做收束，按最终版文章的口径回看 U2 到底交付了什么，以及哪些事情故意没有放进这个 commit。

U2 对应主界面计划 [`docs/plans/2026-06-25-003-feat-first-ink-tui-homepage-plan.md`](https://github.com/kefeiqian/KQode/blob/ad9670d1978ae2c6e738ea76bc52b58149463dbd/docs/plans/2026-06-25-003-feat-first-ink-tui-homepage-plan.md) 的 U2（静态主界面组件）与 U3（输入框输入与换行），也包含主题计划 [`docs/plans/2026-06-29-001-feat-gemini-style-tui-theming-plan.md`](https://github.com/kefeiqian/KQode/blob/ad9670d1978ae2c6e738ea76bc52b58149463dbd/docs/plans/2026-06-29-001-feat-gemini-style-tui-theming-plan.md) 里的半行背景块。代码在 commit [`dd15b678`](https://github.com/kefeiqian/KQode/commit/dd15b678392eacc2ffcee88884eba18ae52c1236)。

## U2 交付了什么

U1 的 [`tui/src/App.tsx`](https://github.com/kefeiqian/KQode/blob/99949b9fe7698a1f0b87acda232281cbaeb4d81d/tui/src/App.tsx) 只是静态空壳，用来证明 Ink 渲染链路能跑。U2 把它推进到一个可交互的终端主界面。

- **启动数据流跑通了**。[`tui/main.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/main.tsx) 采集产品版本、workspace cwd 和 Git 状态，再交给 [`tui/src/App.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/App.tsx)；[`tui/src/App.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/App.tsx) 读取终端尺寸，并把行列数传给主界面。
- **主界面骨架成形了**。[`tui/src/components/HomeScreen.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/HomeScreen.tsx) 纵向组合 Header、正文转录区、cwd 行、输入框和状态栏。[`tui/src/components/layout.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/layout.ts) 负责底部吸附：cwd、输入框、状态栏固定沉到底部，正文和 cwd 之间保留一行空隙。
- **主题和半行背景块变成内部原语**。[`tui/src/theme/themeConfig.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/theme/themeConfig.ts) 集中 GitHub 深色风的语义颜色；[`tui/src/components/BackgroundBlock.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/BackgroundBlock.tsx) 用 `▄` / `▀` 做半行留白，并按 `NO_COLOR`、屏幕阅读器、色深做降级。[第 2 篇](./02-TUI主题与背景块组件.md)把这部分单独展开过。
- **正文区不再只是几行文本**。[`tui/src/components/bodyRows.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/bodyRows.ts) 把类型化的 `BodyEntry` 编译成可渲染行；[`tui/src/components/BodyPane.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/BodyPane.tsx) 处理换行、可见窗口、空行间隔和滚动条。内容溢出时，它会让出最右一列给滚动条，再重新计算正文行，避免滚动条压住文字。
- **cwd 行有了 Git 感知**。[`tui/src/components/CwdLine.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/CwdLine.tsx) 显示 `~` 缩写后的路径，并用 [`tui/src/libs/text/clipText.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/libs/text/clipText.ts) 从左侧裁剪长路径。Git 标签来自 [`tui/src/libs/git/gitStatus.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/libs/git/gitStatus.ts)，解析 `git status --porcelain=v1 --branch`，2 秒还没返回就放弃。
- **输入框可以真正编辑、换行和提交**。[`tui/src/state/composerReducer.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/state/composerReducer.ts) 管文本、光标和校验；[`tui/src/components/PromptComposer.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/PromptComposer.tsx) 接键盘输入、渲染可见窗口、手动定位光标。Shift+Enter 换行，Enter 提交；提交前会挡掉空输入和超过 64 KiB 的 prompt。第 6、7 篇重点补了 Unicode 代理对、UTF-8 字节数、控制字符过滤和 Ink 光标坐标。
- **窄屏和滚动都有明确规则**。[`tui/src/components/Header.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/Header.tsx) 和 [`tui/src/components/StatusBar.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/StatusBar.tsx) 分别按 36 / 52 列、60 列降级；[`tui/src/libs/terminal/mouse.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/libs/terminal/mouse.ts) 解析 SGR 鼠标滚轮，配合 [`tui/src/components/HomeScreen.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/HomeScreen.tsx) 支持滚轮、`PageUp`、`PageDown` 和 `End`。

## 这一版的取舍

U2 没有急着上全局状态。数据还是从 [`tui/main.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/main.tsx) 一路 props 传到 [`tui/src/components/HomeScreen.tsx`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/HomeScreen.tsx)，局部交互用 `useState` 和 `useReducer`。在只有一个屏、状态边界还很清楚的时候，这比提前引入状态容器更直接。等共享状态和 props 链真的开始变重，[U3](/category/u3-jotai-state-refactor) 再迁到 Jotai。

真正容易出错的部分，则尽量拆成纯函数：布局计算在 `resolveHomeScreenLayout`，正文行编译在 [`tui/src/components/bodyRows.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/components/bodyRows.ts)，Git 解析在 [`tui/src/libs/git/gitStatus.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/libs/git/gitStatus.ts)，文本裁剪在 [`tui/src/libs/text/clipText.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/libs/text/clipText.ts)。组件只负责把算好的结果画出来，测试和后续重构都轻一些。

颜色也没有被当成唯一语义来源。用户消息有 `❯`，assistant 行有 `•`，错误有 `ERROR:`，Git 状态有 `*` / `+` / `%`，滚动条有自己的字形。半行背景关掉之后，界面仍然能读。这就是第 2 篇里反复强调的 fallback-aware：终端能力不足时降级，不牺牲内容本身。

另一个取舍是外部环境全部失败软着陆。Git 可能不存在，workspace 可能不是仓库，`git status` 可能卡住；[`tui/src/libs/git/gitStatus.ts`](https://github.com/kefeiqian/KQode/blob/dd15b678392eacc2ffcee88884eba18ae52c1236/tui/src/libs/git/gitStatus.ts) 失败就返回 `undefined`，cwd 行继续显示路径。鼠标追踪也只在 TTY 下开启，退出时再关闭，不把转义序列写进日志或重定向文件。

输入框则把两类后期很难补的细节提前处理掉：一是 Unicode 光标，至少不能把代理对删成 `�`；二是手动光标定位，`composerTop`、可见行号和 Ink 行原点偏移必须一起算。以后只要改正文高度、spacer、输入框换行或校验行，都要重新确认光标还落在当前文本行上。

## 后续边界

U2 只交付可交互的前端外壳。提交 prompt 现在只是追加到正文，再触发外部回调；没有真实后端、没有会话协议，也没有用户可配置主题。这不是遗漏，而是这个 commit 的边界。

后续单元按职责继续往下接：[U3](/category/u3-jotai-state-refactor) 处理 Jotai 状态迁移、组件拆分和 Gemini 风格配色；[U4](/category/u4-ack-backend) 到 [U8](/category/u8-submit-ack-wiring) 接 Rust JSON-RPC 后端、受控启动器、后端客户端和提交 / ACK 回路；[U9](/category/u9-standalone-executable) 到 [U13](/category/u13-release-pipeline) 再做独立可执行文件、架构加固、终端控制、跨平台分发和 Release 流水线。

所以 U2 的重点很明确：先把终端里能看见、能输入、能滚动的主界面做稳。布局、输入、转录和降级路径都站住以后，后面的 agent 循环才有可靠的 UI 承载层。
