`v0.1.1` 是 KQode 从 `v0.1.0` 之后的第一个小版本。它不是一个“大功能一次到位”的版本，而是把早期 TUI、构建环境、版本发布和 Agent 写作工作流补齐成可以继续迭代的形态。

这次发布的核心变化可以概括成四条：

1. TUI 在退出时有了总结卡片，用户离开 alternate screen 之后仍能在正常 scrollback 里看到本次 session 的耗时和工作区改动。
2. TUI 和 Rust core 对 dev / test / prod 的构建标记开始对齐，避免前端和后端各自发明环境判断。
3. `cargo xtask set-version X.Y.Z` 成为版本号的唯一写入口，负责把 Cargo、TUI 和 npm launcher 的 manifest 锁步升级。
4. `.agents/skills` 增加了版本升级 skill，并强化了 blog / xtask / 翻译等作者工作流，让后续发布文档更少依赖临时记忆。

## 文档类 commit 也很重要

虽然本系列不会为每个 docs-only commit 单独开篇，但它们构成了 `v0.1.1` 的上下文。

[`d1813eca`](https://github.com/kefeiqian/KQode/commit/d1813eca6d8219573feeafad10403bb758f71072) 把教程重组为 `U1` / `U2` 章节，并同步中文与英文 locale。这个变化的意义是把早期文章从“线性日志”整理成“可导航的教程单元”，后续版本文章可以更清楚地挂在 release 节点下。

[`d387c78e`](https://github.com/kefeiqian/KQode/commit/d387c78e0c00ced6218c5b0dadf05c371721220f) 新增 LLM provider streaming chat 计划。它没有直接改变 runtime 代码，但提前固定了 provider streaming 的需求边界，避免 TUI 先写死一个无法承载 streaming 的接口。

[`17b51456`](https://github.com/kefeiqian/KQode/commit/17b51456a479ab5de20af403bf1668788099d076) 对 homepage plan、`README.md` 和 repo 级文档做 reconcile。它的价值是让“当前实现”和“产品方向”重新对齐，防止早期项目最容易出现的漂移：代码在往前走，文档还停在旧目标。

[`24ae0934`](https://github.com/kefeiqian/KQode/commit/24ae0934b368e36beb8078c99f53c70c3e71abca) 是发布收口 commit，把版本号推进到 `0.1.1`，并让 release workflow 覆盖新版本。
