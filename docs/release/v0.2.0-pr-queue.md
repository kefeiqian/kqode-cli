# v0.2.0 PR Queue Ledger

This ledger tracks the rolling v0.2.0 split. It is operational state; the plan remains the decision artifact.

## Process Rules

- Open exactly one PR at a time.
- Create extraction worktrees outside the repository root, preferably under a sibling directory such as `../KQode.release-split-worktrees/`.
- Create each branch from freshly fetched `origin/main`.
- Record the base SHA before requesting review.
- Run full CI-equivalent checks before opening a PR.
- Stop for explicit approval before changing queue order or PR boundaries.
- Do not create a generic catch-all polish PR; split leftovers into single-purpose PRs or fold them into a parent PR.
